# Technical Documentation

This folder contains technical documentation for architecture, domain logic, data model, decisions, and operations.

## Core Docs
- `architecture.md` — system architecture and data flows.
- `domain-model.md` — business entities and behavior rules.
- `data-layer.md` — DB/Storage model, policies, and persistence rules.
- `frontend-structure.md` — route and frontend module map.

## Decision Log
- `adr/0001-presentations-storage.md` — accepted decision for presentations persistence.
- `adr/TEMPLATE.md` — template for future ADRs.

## Runbooks
- `runbooks/release-checklist.md` — pre-release and post-deploy checks.
- `runbooks/incident-rollback.md` — incident response and rollback process.
- `runbooks/prod-hotfix-log.md` — mandatory log of manual VPS/production hotfixes.

## Conventions
- Keep documents short and factual.
- Record decisions as ADRs when they affect architecture, data model, or deployment process.
- Update docs when logic, data model, or release flow changes.
